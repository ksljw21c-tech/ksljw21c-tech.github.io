## 수집된 자료 분석
tavily-mcp를 통해 다음과 같은 최신 정보를 수집했습니다:

### 정규화와 비정규화
- 정규화(Normalization): 데이터 중복 최소화, 무결성 보장
- 1NF, 2NF, 3NF 단계별 적용
- 비정규화(Denormalization): 성능을 위한 의도적 중복
- 읽기/쓰기 비율에 따른 전략 선택

### 인덱싱 전략
- 인덱스 생성 기준: WHERE, JOIN, ORDER BY 열
- 복합 인덱스(Composite Index) 활용
- 선택도(Selectivity) 고려한 열 순서
- EXPLAIN을 통한 실행 계획 분석

### 확장성 설계
- 수직적 확장 vs 수평적 확장
- 샤딩(Sharding): 데이터 분산 저장
- 복제(Replication): 읽기 부하 분산
- 캐싱(Caching): Redis, Memcached 활용

### 데이터 무결성과 보안
- 제약 조건: NOT NULL, UNIQUE, FOREIGN KEY, CHECK
- 트랜잭션과 ACID 속성
- 최소 권한 원칙, 데이터 암호화
- SQL 인젝션 방지
- 백업과 복구 전략
