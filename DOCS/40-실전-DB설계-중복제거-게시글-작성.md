# 40. 실전 DB 설계: 중복 제거가 답이다 게시글 작성

- **구조**: H1 p, H2 p, H2 p, H2 p

## 수집된 자료 분석
tavily-mcp를 통해 다음과 같은 최신 정보를 수집했습니다:

### 이상 현상 (Anomaly)
- 삽입 이상 (Insertion Anomaly): 불필요한 정보 함께 입력
- 갱신 이상 (Update Anomaly): 일부만 변경되는 문제
- 삭제 이상 (Deletion Anomaly): 의도치 않은 정보 손실
- 저장 공간 낭비 및 성능 저하

### 정규화 단계
- 제1정규형 (1NF): 원자값만 허용
- 제2정규형 (2NF): 부분 함수 종속 제거
- 제3정규형 (3NF): 이행 함수 종속 제거
- 실무에서는 보통 3NF까지 적용

### 비정규화
- 성능 향상을 위한 의도적 중복
- 읽기 중심 서비스에서 고려
- 정규화 완성 후 선택적 적용
- NoSQL에서는 비정규화가 일반적

### 실전 설계 원칙
- 하나의 테이블에 하나의 개념
- 외래 키를 통한 관계 정의
- 비즈니스 요구사항 반영 (주문단가 등)
- 정규화와 비정규화의 균형
