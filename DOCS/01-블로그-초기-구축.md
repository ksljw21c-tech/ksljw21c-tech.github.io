## ✨ 주요 기능

1. **마크다운 렌더링**: marked.js를 사용하여 마크다운을 HTML로 변환
2. **코드 하이라이팅**: Prism.js로 다양한 언어 코드 하이라이팅
3. **다크/라이트 모드**: CSS 변수 기반, 시스템 설정 자동 감지
4. **검색 기능**: 제목, 내용, 태그 검색 지원
5. **태그 필터링**: 태그별 게시글 필터링
6. **댓글 시스템**: Giscus (GitHub Discussions 기반)
7. **자동 배포**: Git push 시 GitHub Actions가 자동 배포

## 🎨 디자인 특징

- **색상 테마**: 따뜻한 오렌지 계열 악센트 색상
- **타이포그래피**: 가독성 좋은 Pretendard 폰트 (시스템 폰트 폴백)

## ⚠️ 다음 단계 (필수)

### 1. GitHub 저장소 생성
```bash
# GitHub에서 ksljw21c-tech.github.io 저장소 생성 후
git remote add origin https://github.com/ksljw21c-tech/ksljw21c-tech.github.io.git
```

### 2. GitHub Pages 설정
1. 저장소 **Settings** → **Pages**
2. **Source**: "GitHub Actions" 선택

### 3. 첫 배포
```bash
git add .
git commit -m "feat: 블로그 초기 구축"
git push origin main
```

### 4. Giscus 댓글 설정 (선택)
1. 저장소 Settings → Features → **Discussions** 활성화
2. https://github.com/apps/giscus 에서 앱 설치
3. https://giscus.app/ko 에서 설정 정보 가져오기
4. `js/post-loader.js`의 `loadGiscus()` 함수에서 `data-repo-id`와 `data-category-id` 수정

## 📝 새 글 작성 방법

1. `pages/` 폴더에 새 `.md` 파일 생성
2. Front Matter 작성:
```yaml
---
title: '글 제목'
date: 2025-01-26
tags: ['태그1', '태그2']
category: '카테고리'
description: '글 설명'
---
```
3. 마크다운으로 본문 작성
4. Git commit & push
5. 자동 배포 완료!
