tavily-mcp를 통해 다음과 같은 최신 정보를 수집했습니다:

### RDBMS 기본 개념
- 관계형 데이터베이스의 역사 (에드거 코드, 1970년)
- Oracle, MySQL, PostgreSQL 등 주요 RDBMS
- 데이터 일관성과 무결성의 중요성
- NoSQL과의 차이점

### 설계 프로세스
- 요구사항 분석: 엔티티, 속성, 관계 식별
- 개념적 설계: 비즈니스 관점의 데이터 모델
- 논리적 설계: 관계형 모델로 변환, 정규화
- 물리적 설계: DBMS 구현, 성능 최적화

### ERD 작성
- 엔티티: 사각형으로 표현
- 속성: 엔티티 내 나열
- 관계: 선으로 연결, 기수성 표시
- N:M 관계 처리: 연결 테이블 사용
- 도구: MySQL Workbench, ERDCloud, draw.io

### 정규화와 무결성
- 1NF, 2NF, 3NF 정규화 단계
- 개체 무결성, 참조 무결성, 도메인 무결성
- 제약조건: PK, FK, NOT NULL, UNIQUE, CHECK

### 물리적 설계
- 인덱스 전략
- 파티셔닝 기법
- 반정규화 고려
- DDL 스크립트 관리
