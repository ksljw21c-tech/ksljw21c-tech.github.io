# 51. 온라인 강의 플랫폼 데이터베이스 구축 게시글 작성

## 작업 개요
- **주제**: 온라인 강의 플랫폼 DB 설계(핵심 엔티티·관계·운영 체크리스트)
- **구조**: H1 p, H2 p, H2 p, H2 p (서술형)
- **목표 글자수**: 5,000자 / **실제**: 5,100자
- **용도**: 애드센스 승인용 고품질 콘텐츠
- **도구**: tavily-mcp 최신 자료 수집

## 수집된 자료 분석
- GeeksforGeeks 온라인 러닝 플랫폼 ERD: User–Course N:M, Payment 1:N, Progress 테이블 분리 예시
- 상용 플랫폼 기능 비교(Udemy/Thinkific 등 소개 글): 카테고리/태그, 리뷰, 정산, 쿠폰 기능 강조
- 일반 베스트 프랙티스: N:M은 연결 테이블, 환불 상태 기록, 인덱스·캐싱·파티셔닝 활용

## 작성된 글 핵심
- 핵심 엔티티: users, courses, lessons, enrollments, progress, payments, reviews, categories(+course_categories)
- 관계 설계: Course–Lesson 1:N, User–Course N:M→enrollments, User–Payment 1:N, Course–Category N:M
- 스키마 포인트: 강사 프로필 분리, enrollments 복합 PK/유니크, progress 레슨 단위 기록, refunds/payouts 선택 추가
- 운영 체크리스트: 인덱스·캐싱·파티셔닝, FK 무결성 정책 문서화, 보안(해시·토큰화·암호화), 트랜잭션/큐, 모니터링, 백업·복구

## 글자 수
- 5,100자 (목표 충족)
