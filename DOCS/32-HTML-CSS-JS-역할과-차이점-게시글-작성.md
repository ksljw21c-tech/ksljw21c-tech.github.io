### Front Matter
- **제목**: "HTML CSS JS 역할과 차이점" (15자, 25자 이내)

### 본문 구조

1. **H1: 제목 및 전체 개요**
   - HTML, CSS, JavaScript의 중요성
   - 각 기술의 독특한 역할
   - 집 건축 비유 (건축가=HTML, 디자이너=CSS, 기술자=JavaScript)
   - 웹 개발의 기초
   - 글의 목적과 구성 설명

2. **H2-1: HTML이란 무엇인가 - 웹페이지의 뼈대와 구조**
   - HTML의 정의
     - HyperText Markup Language의 약자
     - 웹페이지의 구조를 만드는 마크업 언어
     - 뼈대나 골격 역할
   - HTML의 비유
     - 집 건축 (벽, 기둥, 문, 창문 배치)
     - 인체의 뼈대
   - HTML의 역사
     - 1991년 팀 버너스 리가 개발
     - HTML5가 현재 표준
     - 의미론적 요소 제공
   - HTML의 핵심 개념
     - 태그(Tag) 개념
     - 계층 구조와 트리 구조
     - 의미론적(Semantic) 태그
   - HTML의 주요 역할
     - 콘텐츠의 구조화
     - 다양한 미디어 콘텐츠 포함
     - 논리적이고 의미 있는 구조 배치
   - HTML의 한계
     - 스타일이나 디자인 정보 미포함
     - CSS와 JavaScript와의 결합 필요
   - HTML의 중요성
     - 모든 웹페이지의 기초
     - 다른 모든 웹 기술의 기반
     - 웹 개발의 첫 번째 단계

3. **H2-2: CSS란 무엇인가 - 웹페이지의 디자인과 스타일**
   - CSS의 정의
     - Cascading Style Sheets의 약자
     - 웹페이지의 디자인과 시각적 스타일 담당
     - 구조를 아름답고 매력적으로 꾸미는 역할
   - CSS의 비유
     - 집 인테리어 (벽지, 가구, 조명)
     - 인체의 피부
   - CSS의 역사
     - 1996년 처음 소개
     - CSS3가 현재 표준
   - CSS의 핵심 개념
     - 선택자(Selector)와 속성(Property)
     - 다양한 선택자 (클래스, ID, 속성 선택자)
     - 상속(Inheritance)과 캐스케이딩(Cascading)
   - CSS의 주요 역할
     - 시각적 디자인 (색상, 글꼴, 크기, 간격, 배경)
     - 레이아웃 구성 (Flexbox, Grid)
     - 반응형 디자인 (미디어 쿼리)
   - CSS의 장점
     - HTML과 분리
     - 유지보수 용이
     - 일관된 디자인 유지
     - 애니메이션과 전환 효과
   - CSS의 한계
     - 정적인 스타일만 제공
     - 사용자 상호작용에 제한적
     - JavaScript 필요

4. **H2-3: JavaScript란 무엇인가 - 웹페이지의 상호작용과 동적 기능**
   - JavaScript의 정의
     - 웹페이지에 상호작용과 동적 기능 추가
     - 프로그래밍 언어
     - 웹페이지를 살아 숨 쉬게 만드는 역할
   - JavaScript의 비유
     - 집의 전기와 배관
     - 인체의 신경계
   - JavaScript의 역사
     - 1995년 브렌단 아이크가 개발
     - 웹 개발의 핵심 언어
   - JavaScript의 주요 특징
     - 클라이언트 사이드에서 실행
     - 브라우저에서 직접 실행
     - 실시간 콘텐츠 변경
   - JavaScript의 핵심 기능
     - DOM 조작
     - 이벤트 처리
     - 비동기 통신 (AJAX)
   - JavaScript의 활용
     - 사용자 상호작용 (클릭, 스크롤, 입력)
     - 폼 검증
     - 애니메이션 실행
     - 서버와 데이터 교환
   - JavaScript의 확장
     - Node.js (서버 사이드)
     - React, Vue.js, Angular (프레임워크)
     - 모바일 앱, 데스크톱 앱, 게임

5. **H2-4: HTML, CSS, JavaScript의 관계와 협력 방식**
   - 세 기술의 독립성과 협력
     - 서로 독립적이지만 함께 작동
     - 완전한 웹 경험 창조
   - HTML의 기초 역할
     - 세 가지 기술의 기초
     - CSS와 JavaScript의 대상
   - 관심사의 분리
     - 각 기술이 자신의 역할에 집중
     - 유지보수 용이
   - 연결 방식의 차이
     - CSS: `<link>` 태그 또는 `<style>` 태그
     - JavaScript: `<script>` 태그
   - 상호 보완
     - HTML: 구조 제공, 디자인과 상호작용 부족
     - CSS: 아름다운 디자인, 정적
     - JavaScript: 동적 기능, 복잡한 디자인 어려움
   - 협력 예시
     - 버튼 만들기 (HTML) + 디자인 (CSS) + 동작 (JavaScript)

6. **H2-5: 실전 예시로 보는 HTML, CSS, JavaScript의 협력**
   - 로그인 폼 예시
     - HTML: 폼 구조 (`<form>`, `<input>`, `<button>`)
     - CSS: 디자인 (색상, 크기, 간격, 테두리)
     - JavaScript: 기능 (입력 검증, 제출 처리, 에러 메시지)
   - 현대적 웹 개발 표준
     - 세 가지 기술의 균형 잡힌 사용
     - 프론트엔드 개발자의 필수 지식
   - 각 기술의 역할
     - HTML: 콘텐츠의 의미 구조화
     - CSS: 사용자 경험 향상
     - JavaScript: 복잡한 로직 처리
   - 2025년 웹 개발 트렌드
     - HTML5, CSS3, ES6+ JavaScript 표준화
     - HTML과 CSS의 기능 확장으로 JavaScript 의존도 감소
     - 프레임워크와 라이브러리의 효율성
   - 기본 원리의 불변성
     - HTML: 구조
     - CSS: 스타일
     - JavaScript: 상호작용
   - 웹 개발의 기초
     - 모든 웹 개발자의 필수 지식
     - 프레임워크 이해의 기반
     - 웹 개발자로서의 성공적인 여정의 시작
