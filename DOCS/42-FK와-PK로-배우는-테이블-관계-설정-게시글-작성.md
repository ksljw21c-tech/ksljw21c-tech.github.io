### 기본키(PK) 개념
- 각 행을 고유하게 식별하는 열
- 유일성(Uniqueness)과 NOT NULL 조건
- 자연키 vs 대리키 선택
- AUTO_INCREMENT 활용
- 복합키(Composite Key) 개념

### 외래키(FK) 개념
- 다른 테이블의 PK를 참조하는 열
- 부모-자식 테이블 관계
- 참조 무결성 보장
- REFERENCES 구문 사용
- ON DELETE, ON UPDATE 옵션

### 관계 유형
- 1:1 관계: UNIQUE + FK 또는 PK 공유
- 1:N 관계: N쪽에 FK 배치
- N:M 관계: 연결 테이블(Junction Table) 사용
- 복합 FK를 통한 관계 구현

### 실전 고려사항
- 테이블 생성 순서 (부모 먼저)
- 식별 관계 vs 비식별 관계
- FK 인덱스 설정 중요성
- 삭제 정책 (CASCADE, RESTRICT, SET NULL)
- 순환 참조 회피
