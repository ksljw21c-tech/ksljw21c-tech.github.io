## 📝 게시글 내용 구성

### Front Matter
- **제목**: "웹의 3대 구성 요소 완벽 정리" (15자, 25자 이내)

- **카테고리**: '개발'
- **설명**: 웹 개발의 기초가 되는 HTML, CSS, JavaScript 세 가지 핵심 기술의 역할과 협력 관계를 완벽하게 정리

### 본문 구조

1. **H1: 제목 및 전체 개요**
   - 웹의 3대 구성 요소: HTML, CSS, JavaScript
   - 모든 웹사이트와 웹 애플리케이션의 기초
   - 각 기술의 역할 소개
   - 집 건축 비유 (HTML=골격, CSS=인테리어, JavaScript=기능)
   - 글의 목적과 구성 설명

2. **H2-1: HTML - 웹의 구조와 골격을 만드는 마크업 언어**
   - HTML의 정의와 역할
     - HyperText Markup Language
     - 마크업 언어 (프로그래밍 언어 아님)
     - 웹페이지의 구조와 내용 정의
   - HTML의 기본 기능
     - 제목, 단락, 이미지, 링크, 폼 등 콘텐츠 요소 정의
     - 태그를 통한 구조화
   - HTML5와 의미론적 태그
     - `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<footer>`
     - 접근성과 SEO 향상
   - 하이퍼링크의 중요성
     - 웹페이지 간 연결
     - 웹의 핵심 기능
   - 폼 요소
     - 사용자 입력 받기
     - 다양한 입력 요소 (텍스트, 체크박스, 라디오, 드롭다운)
   - 2025년 HTML의 발전
     - 웹 컴포넌트, 웹 접근성, 모바일 최적화
   - HTML의 기초 역할
     - CSS와 JavaScript의 기반
     - 웹 개발의 첫 단계

3. **H2-2: CSS - 웹의 스타일과 디자인을 담당하는 스타일시트**
   - CSS의 정의와 역할
     - Cascading Style Sheets
     - 시각적 스타일과 레이아웃 담당
   - 내용과 스타일의 분리
     - HTML은 구조, CSS는 표현
     - 재사용성과 유지보수성 향상
   - CSS의 기본 작동 방식
     - 선택자(Selector)를 통한 요소 선택
     - 속성(Property)과 값(Value)으로 스타일 적용
   - CSS3의 주요 기능
     - Flexbox와 Grid 레이아웃
     - 애니메이션과 트랜지션
     - 미디어 쿼리 (반응형 웹 디자인)
   - 2025년 CSS의 최신 기능
     - Container Queries
     - Logical Properties
     - 새로운 색상 함수
     - CSS 변수 (Custom Properties)
     - 다크 모드 지원
   - CSS의 사용자 경험 영향
     - 외관 결정
     - 정보 접근성 향상
     - 시각적 경험 제공

4. **H2-3: JavaScript - 웹의 상호작용과 동적 기능을 구현하는 프로그래밍 언어**
   - JavaScript의 정의와 역할
     - 동적 기능과 상호작용 추가
     - 웹의 95% 이상에서 사용
   - JavaScript의 역사와 발전
     - 1995년 등장
     - 웹뿐만 아니라 서버, 모바일, 데스크톱에서도 사용
   - DOM 조작
     - HTML 요소 선택 및 수정
     - 내용 변경, 스타일 수정, 이벤트 처리
   - 비동기 프로그래밍
     - fetch API, XMLHttpRequest
     - Promise, async/await
     - 페이지 새로고침 없이 콘텐츠 업데이트
   - JavaScript의 현대적 기능
     - ES2025 표준
     - 모듈 시스템, 클래스, 화살표 함수
     - 구조 분해 할당, 템플릿 리터럴
   - Web APIs
     - Local Storage, Geolocation
     - Web Workers, Service Workers
   - JavaScript의 중요성
     - 정적 페이지를 동적 애플리케이션으로 변환
     - 모든 상호작용 기능 구현

5. **H2-4: 세 가지 기술의 협력과 실전 활용 방법**
   - 역할 분담의 원칙
     - HTML: 구조와 내용
     - CSS: 스타일과 레이아웃
     - JavaScript: 동작과 상호작용
   - 관심사의 분리
     - 코드 유지보수성 향상
     - 팀 협업 용이
     - 전문성 발휘
   - 실전 협력 예제
     - 로그인 폼 예제
     - 각 기술의 역할 설명
   - 2025년 통합 트렌드
     - CSS `:has()` 선택자
     - 웹 컴포넌트
     - 모던 프레임워크의 컴포넌트 단위 관리
     - 빌드 도구의 최적화
   - 기본 원리의 중요성
     - 프레임워크는 세 가지 기술의 추상화
     - 문제 해결을 위한 근본 원리 이해
   - 순수 기술의 가치
     - 빠르고 가벼운 웹사이트
     - 복잡한 빌드 과정 없이도 강력한 애플리케이션
   - 학습 순서의 중요성
     - 프레임워크보다 기본 기술 먼저
     - 튼튼한 기초의 필요성
