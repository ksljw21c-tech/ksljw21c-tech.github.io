### 본문 구조

1. **H1: 제목 및 전체 개요**
   - 피드백의 중요성
   - 비침투적 알림의 필요성
   - 스낵바와 토스트 메시지의 역할
   - 글의 목적과 구성 설명

2. **H2-1: 스낵바와 토스트 메시지의 차이점**
   - 스낵바의 특성
     - 화면 하단 배치
     - 액션 버튼 포함 가능
     - 4-10초 지속
     - 여러 개 쌓일 수 있음
   - 토스트 메시지의 특성
     - 화면 모서리 배치
     - 액션 버튼 없음
     - 3-5초 지속
     - 여러 개 쌓일 수 있음
   - 각각의 사용 목적

3. **H2-2: 스낵바 디자인 원칙**
   - 명확한 메시지
   - 액션 버튼
   - 위치 (화면 하단 중앙)
   - 시각적 디자인
     - 대비
     - 패딩과 여백
     - 애니메이션
   - 닫기 버튼

4. **H2-3: 토스트 메시지 디자인 원칙**
   - 간결성 (한 줄 또는 두 줄)
   - 위치 (상단 오른쪽)
   - 지속 시간 (3-5초)
   - 시각적 디자인
     - 대비
     - 아이콘
     - 색상 구분
   - 여러 개 쌓기

5. **H2-4: 비침투적 디자인의 핵심**
   - 자동 사라짐
   - 모달이 아닌 디자인
   - 적절한 크기
   - 타이밍
   - 애니메이션 신중 사용

6. **H2-5: 모바일 환경 최적화**
   - 화면 하단/상단 전체 너비 사용
   - 터치 제스처 지원
   - 터치 친화적 디자인
   - 한 번에 하나의 스낵바
   - 키보드 대응

7. **H2-6: 접근성 고려사항**
   - 스크린 리더 지원 (aria-live)
   - 키보드 접근성
   - 색상에만 의존하지 않기
   - 충분한 대비
   - prefers-reduced-motion 지원

8. **H2-7: 사용 시점과 주의사항**
   - 스낵바 사용 시점
   - 토스트 메시지 사용 시점
   - 중요한 오류는 모달 사용
   - 과도한 사용 피하기
   - 일관성 유지
   - 지속적인 정보에는 부적합
