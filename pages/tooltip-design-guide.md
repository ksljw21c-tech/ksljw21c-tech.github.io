# 툴팁 디자인 가이드 - 사용자 경험을 해치지 않는 보조 안내 도구 활용법

웹사이트나 애플리케이션을 사용하다 보면 특정 요소에 마우스를 올리거나 포커스할 때 작은 정보 박스가 나타나는 것을 경험합니다. 이것이 툴팁입니다. 툴팁은 작은 UI 요소이지만 사용자 경험에 큰 영향을 미칩니다. 잘 설계된 툴팁은 사용자에게 필요한 정보를 적시에 제공하여 혼란을 줄이고 작업 효율을 높입니다. 반면, 잘못 설계된 툴팁은 사용자를 방해하고 불편함을 느끼게 할 수 있습니다. 이 글에서는 툴팁 디자인의 핵심 원칙과 베스트 프랙티스를 상세히 살펴보겠습니다. 툴팁은 특정 UI 요소에 대한 추가 정보나 컨텍스트를 제공하는 작은 오버레이 요소입니다. 툴팁은 사용자의 작업 흐름을 방해하지 않으면서 필요한 정보를 제공하는 것이 목적입니다. 툴팁을 효과적으로 사용하려면 적절한 시점에 적절한 정보를 제공해야 하며, 접근성과 사용성을 모두 고려해야 합니다. 이러한 원칙들을 이해하고 적용하면 사용자 경험을 해치지 않으면서도 유용한 안내를 제공할 수 있습니다.

## 툴팁의 정의와 역할

툴팁은 특정 UI 요소에 대한 추가 정보나 컨텍스트를 제공하는 작은 오버레이 요소입니다. 툴팁은 사용자가 특정 요소에 호버하거나 포커스할 때 나타나며, 해당 요소에 대한 설명이나 안내를 제공합니다. 툴팁의 가장 중요한 특징은 비침투성입니다. 툴팁은 사용자의 작업 흐름을 방해하지 않으면서 필요한 정보를 제공하며, 사용자가 원할 때만 나타나고 쉽게 사라질 수 있습니다. 툴팁은 일반적으로 아이콘, 버튼, 링크 등과 같은 UI 요소에 연결되어 있으며, 해당 요소의 기능이나 의미를 설명합니다. 예를 들어, 아이콘만 있는 버튼에 툴팁을 추가하면 사용자가 버튼의 기능을 이해할 수 있습니다. 툴팁은 또한 복잡한 인터페이스에서 사용자에게 안내를 제공하는 데 유용합니다.

툴팁의 역할은 크게 세 가지로 나눌 수 있습니다. 첫째, 명확성 제공입니다. 툴팁은 모호하거나 불명확한 UI 요소에 대한 설명을 제공하여 사용자가 요소의 기능을 이해할 수 있게 해줍니다. 둘째, 컨텍스트 제공입니다. 툴팁은 특정 상황에서 필요한 추가 정보를 제공하여 사용자가 더 나은 결정을 내릴 수 있게 해줍니다. 셋째, 학습 지원입니다. 툴팁은 사용자가 새로운 기능을 배우거나 복잡한 인터페이스를 이해하는 데 도움을 줍니다. 이러한 역할을 효과적으로 수행하려면 툴팁은 간결하고 명확해야 하며, 사용자가 쉽게 이해할 수 있어야 합니다. 툴팁은 필수 정보를 포함해서는 안 되며, 필수 정보는 항상 표시되어야 합니다. 툴팁은 보조적인 정보를 제공하는 도구이므로, 사용자가 툴팁을 보지 않아도 기본 작업을 수행할 수 있어야 합니다.

## 툴팁의 트리거 방식과 타이밍

툴팁은 다양한 방식으로 트리거될 수 있으며, 각 방식은 장단점이 있습니다. 가장 일반적인 트리거 방식은 호버입니다. 데스크톱 환경에서 마우스를 특정 요소 위에 올리면 툴팁이 나타나며, 마우스를 떼면 사라집니다. 호버 트리거는 사용자가 자연스럽게 정보를 얻을 수 있게 해주지만, 모바일 환경에서는 작동하지 않습니다. 포커스 트리거는 키보드 사용자를 위해 중요합니다. 사용자가 키보드로 특정 요소에 포커스하면 툴팁이 나타나며, 포커스를 잃으면 사라집니다. 포커스 트리거는 접근성을 위해 필수적이며, 모든 툴팁은 호버와 포커스 모두에서 작동해야 합니다. 클릭 트리거는 모바일 환경에서 유용합니다. 모바일 환경에서는 호버가 없으므로, 사용자가 요소를 탭하면 툴팁이 나타나고, 다른 곳을 탭하면 사라집니다.

툴팁의 타이밍도 중요합니다. 툴팁이 너무 빨리 나타나면 사용자가 의도하지 않았을 때 방해가 될 수 있으며, 너무 늦게 나타나면 사용자가 기다리게 됩니다. 일반적으로 호버 후 0.5-1초 정도의 지연이 적절하며, 이는 사용자가 의도적으로 정보를 찾을 때만 툴팁이 나타나도록 합니다. 툴팁이 사라지는 타이밍도 중요합니다. 사용자가 툴팁을 읽을 충분한 시간을 제공해야 하며, 일반적으로 마우스를 떼거나 포커스를 잃은 후 즉시 사라지는 것이 적절합니다. 하지만 사용자가 툴팁 위에 마우스를 올릴 수 있도록 하면 더 나은 사용자 경험을 제공할 수 있습니다. 툴팁의 지속 시간도 고려해야 합니다. 사용자가 툴팁을 읽을 충분한 시간을 제공하되, 너무 오래 남아있어서 방해가 되지 않도록 해야 합니다.

## 툴팁 디자인 원칙

툴팁을 효과적으로 디자인하려면 몇 가지 핵심 원칙을 따라야 합니다. 첫째, 간결성입니다. 툴팁은 짧고 명확한 메시지를 포함해야 하며, 일반적으로 한 줄 또는 두 줄을 넘지 않아야 합니다. 긴 텍스트는 사용자가 읽기 어렵고, 툴팁의 목적에 맞지 않습니다. 둘째, 명확성입니다. 툴팁의 메시지는 명확하고 이해하기 쉬워야 하며, 기술적인 용어나 전문 용어는 피해야 합니다. 사용자가 즉시 이해할 수 있는 언어를 사용해야 합니다. 셋째, 위치입니다. 툴팁은 관련 요소 근처에 배치되어야 하며, 요소를 가리지 않아야 합니다. 일반적으로 툴팁은 요소의 위, 아래, 왼쪽, 오른쪽에 배치될 수 있으며, 화면 공간에 따라 자동으로 조정되어야 합니다.

툴팁의 시각적 디자인도 중요합니다. 툴팁은 배경과 충분한 대비를 가져야 하며, 텍스트는 읽기 쉬워야 합니다. 툴팁은 화살표나 포인터를 사용하여 관련 요소를 가리킬 수 있으며, 이는 툴팁과 요소의 연결을 시각적으로 명확하게 합니다. 툴팁의 색상과 스타일은 애플리케이션의 전반적인 디자인과 일관성을 유지해야 하며, 너무 눈에 띄거나 방해가 되는 디자인은 피해야 합니다. 툴팁은 또한 적절한 패딩과 여백을 가져야 하며, 텍스트가 답답하지 않도록 충분한 공간을 제공해야 합니다. 툴팁의 크기도 중요합니다. 툴팁은 내용에 맞는 크기여야 하며, 너무 크거나 작으면 사용자 경험에 영향을 미칩니다.

## 접근성 고려사항

툴팁은 모든 사용자가 접근할 수 있어야 하며, 접근성을 고려하지 않은 툴팁은 많은 사용자에게 유용하지 않습니다. 첫째, 키보드 접근성입니다. 모든 툴팁은 키보드 포커스에서도 작동해야 하며, 호버만으로는 충분하지 않습니다. 키보드 사용자가 요소에 포커스하면 툴팁이 나타나야 하며, 포커스를 잃으면 사라져야 합니다. 둘째, 스크린 리더 지원입니다. 툴팁은 ARIA 속성을 사용하여 스크린 리더 사용자가 툴팁의 내용을 이해할 수 있도록 해야 합니다. `aria-describedby` 속성을 사용하여 요소와 툴팁을 연결할 수 있으며, 이를 통해 스크린 리더가 툴팁의 내용을 읽을 수 있습니다. 셋째, 충분한 시간 제공입니다. 툴팁은 사용자가 읽을 충분한 시간을 제공해야 하며, 너무 빨리 사라지면 사용자가 내용을 이해할 수 없습니다.

툴팁의 접근성은 WCAG 가이드라인을 따라야 합니다. WCAG 1.4.13은 호버나 포커스로 트리거되는 콘텐츠에 대한 가이드라인을 제공하며, 툴팁은 이 가이드라인을 준수해야 합니다. 툴팁은 닫을 수 있어야 하며, 사용자가 툴팁을 닫을 수 있는 방법을 제공해야 합니다. 툴팁은 호버 가능해야 하며, 사용자가 툴팁 위에 마우스를 올릴 수 있어야 합니다. 툴팁은 지속적이어야 하며, 사용자가 마우스를 떼거나 포커스를 잃을 때까지 유지되어야 합니다. 이러한 접근성 고려사항을 충족하면 모든 사용자가 툴팁을 효과적으로 사용할 수 있습니다.

## 사용 시점과 주의사항

툴팁은 적절한 시점에 사용해야 하며, 모든 상황에 적합한 것은 아닙니다. 툴팁은 보조적인 정보를 제공하는 데 적합하며, 필수 정보는 항상 표시되어야 합니다. 툴팁은 사용자가 특정 요소의 기능을 이해하는 데 도움이 될 때 사용해야 하며, 예를 들어 아이콘만 있는 버튼, 축약된 텍스트, 전문 용어 등에 유용합니다. 툴팁은 또한 사용자가 새로운 기능을 배우거나 복잡한 인터페이스를 이해하는 데 도움이 될 때 사용할 수 있습니다. 하지만 툴팁은 남용해서는 안 됩니다. 너무 많은 툴팁은 사용자를 방해하고 혼란스럽게 만들 수 있으며, 인터페이스가 불명확하다는 신호일 수 있습니다.

툴팁을 사용할 때 주의해야 할 사항들이 있습니다. 첫째, 툴팁 내에 상호작용 요소를 포함하지 않아야 합니다. 툴팁은 정보를 제공하는 용도이므로, 버튼이나 링크와 같은 상호작용 요소를 포함하면 접근성 문제가 발생할 수 있습니다. 둘째, 툴팁이 다른 중요한 요소를 가리지 않아야 합니다. 툴팁은 관련 요소 근처에 배치되어야 하며, 다른 중요한 버튼이나 콘텐츠를 가리지 않아야 합니다. 셋째, 툴팁의 일관성을 유지해야 합니다. 모든 툴팁은 동일한 스타일과 동작을 가져야 하며, 사용자가 예측 가능하게 사용할 수 있어야 합니다. 넷째, 툴팁은 필수 정보를 포함해서는 안 됩니다. 필수 정보는 항상 표시되어야 하며, 툴팁에만 의존하면 안 됩니다.

## 모바일 환경 고려사항

모바일 환경에서는 툴팁의 사용이 더욱 신중해야 합니다. 모바일 환경에서는 호버가 없으므로, 툴팁은 탭으로 트리거되어야 합니다. 사용자가 요소를 탭하면 툴팁이 나타나고, 다른 곳을 탭하면 사라져야 합니다. 모바일 환경에서 툴팁은 화면 크기를 고려하여 적절한 크기로 표시되어야 하며, 작은 화면에서도 읽기 쉬워야 합니다. 모바일 환경에서 툴팁은 또한 터치 타겟을 고려해야 하며, 사용자가 쉽게 탭할 수 있는 크기와 위치에 있어야 합니다. 모바일 환경에서는 툴팁 대신 항상 표시되는 라벨이나 인라인 도움말을 사용하는 것이 더 나을 수 있습니다.

모바일 환경에서 툴팁을 사용할 때는 사용자가 툴팁을 쉽게 닫을 수 있어야 하며, 툴팁이 화면을 너무 많이 차지하지 않아야 합니다. 모바일 환경에서는 툴팁이 화면의 상단이나 하단에 배치되는 것이 좋으며, 중앙에 배치되면 다른 콘텐츠를 가릴 수 있습니다. 모바일 환경에서 툴팁은 또한 스와이프 제스처로 닫을 수 있도록 하면 더 나은 사용자 경험을 제공할 수 있습니다. 모바일 환경에서는 툴팁의 사용을 최소화하고, 가능하면 항상 표시되는 정보를 제공하는 것이 더 나을 수 있습니다.
